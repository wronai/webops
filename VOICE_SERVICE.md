# NLP2CMD Voice Service

Complete voice-controlled command execution service running in Docker with real-time log streaming and web interface.

## 🎯 Features

- **🎤 Voice Control** - Speech-to-text command input (placeholder for STT integration)
- **⌨️ Text Input** - Fallback text command input
- **🔧 Shell Execution** - Execute generated commands in Docker container
- **📊 Real-time Logs** - Live log streaming via WebSocket
- **🌐 Web Interface** - Modern responsive web UI
- **🐳 Docker Ready** - Complete containerized deployment
- **🔒 Secure** - Non-root user, rate limiting, CORS protection
- **📡 WebSocket** - Real-time communication
- **🔄 Auto-reconnect** - Automatic WebSocket reconnection

## 🚀 Quick Start

### 1. Setup and Run

```bash
# Clone and setup
git clone <repository>
cd nlp2cmd

# Run the setup script
./setup-voice-service.sh

# Or manual setup:
docker build -f Dockerfile.voice -t nlp2cmd-voice:latest .
docker-compose -f docker-compose.voice.yml up -d
```

### 2. Access the Service

Open your browser and go to: **http://localhost:8000**

### 3. Use Voice Commands

- **Voice**: Click "🎤 Nagraj komendę" and speak clearly
- **Text**: Type command and click "▶️ Wykonaj"
- **Examples**:
  - "list files in current directory"
  - "show system processes" 
  - "find files larger than 100MB"
  - "check disk space usage"

## 🔧 Configuration

### Environment Variables

```bash
# Service Configuration
NLP2CMD_HOST=0.0.0.0
NLP2CMD_PORT=8000
NLP2CMD_DEBUG=false
NLP2CMD_LOG_LEVEL=info
NLP2CMD_AUTO_EXECUTE=true

# Voice Service (future)
VOICE_STT_ENGINE=whisper
VOICE_TTS_ENGINE=elevenlabs
VOICE_LANGUAGE=pl
```

### Docker Compose Options

```yaml
services:
  nlp2cmd-voice:
    environment:
      - NLP2CMD_AUTO_EXECUTE=true  # Auto-execute commands
      - NLP2CMD_DEBUG=false        # Debug mode
    volumes:
      - ./logs:/app/logs           # Persistent logs
      - /var/run/docker.sock:/var/run/docker.sock  # Docker commands
    ports:
      - "8000:8000"               # API and web interface
```

## 📊 API Endpoints

### REST API
- `GET /` - Web interface
- `GET /health` - Health check
- `POST /voice-command` - Process voice/text command

### WebSocket
- `WS /ws` - Real-time log streaming

### Request/Response Format

```json
{
  "audio_data": "base64-encoded-audio",
  "text_command": "list files in current directory", 
  "language": "pl",
  "execute": true
}

{
  "success": true,
  "command": "ls -la",
  "explanation": "Generated by RuleBasedPipeline with confidence 0.95",
  "confidence": 0.95,
  "execution_result": {
    "success": true,
    "exit_code": 0,
    "stdout": "total 12\ndrwxr-xr-x 3 user user 4096 Jan 25 11:30 .",
    "stderr": "",
    "logs": ["total 12", "drwxr-xr-x 3 user user 4096 Jan 25 11:30 ."]
  }
}
```

## 🛡️ Security Features

- **Non-root User**: Container runs as non-root user
- **Rate Limiting**: Nginx rate limiting for API/WebSocket
- **CORS Protection**: Configurable CORS origins
- **Input Validation**: Pydantic model validation
- **Command Timeout**: 30-second execution timeout
- **Log Isolation**: Commands executed in controlled environment

### API Usage
```python
import requests

# Send voice command
response = requests.post('http://localhost:8000/voice-command', json={
    'text_command': 'list files',
    'language': 'pl',
    'execute': True
})

result = response.json()
print(f"Command: {result['command']}")
print(f"Logs: {result['execution_result']['logs']}")
```

### WebSocket Client
```javascript
const ws = new WebSocket('ws://localhost:8000/ws');

ws.onmessage = function(event) {
    const data = JSON.parse(event.data);
    if (data.type === 'log') {
        console.log('Log:', data.message);
    }
};
```

## 🚀 Production Deployment

### SSL/HTTPS Setup
```bash
# Generate SSL certificates
mkdir -p nginx/ssl
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout nginx/ssl/key.pem \
  -out nginx/ssl/cert.pem

# Enable HTTPS in docker-compose
docker-compose -f docker-compose.voice.yml --profile production up -d
```

### Scaling
```yaml
# docker-compose.voice.yml
services:
  nlp2cmd-voice:
    deploy:
      replicas: 3
    environment:
      - NLP2CMD_WORKERS=4
```

## 🔧 Troubleshooting

### Common Issues

**Service won't start:**
```bash
# Check Docker logs
docker logs nlp2cmd-voice

# Check port conflicts
netstat -tlnp | grep :8000

# Rebuild image
docker build -f Dockerfile.voice --no-cache -t nlp2cmd-voice:latest .
```

**WebSocket connection issues:**
```bash
# Check nginx configuration
docker exec nlp2cmd-nginx nginx -t

# Restart services
docker-compose -f docker-compose.voice.yml restart
```

**Command execution failures:**
```bash
# Check container permissions
docker exec nlp2cmd-voice whoami

# Test shell access
docker exec -it nlp2cmd-voice /bin/bash
```

### Debug Mode
```bash
# Enable debug logging
docker-compose -f docker-compose.voice.yml down
docker-compose -f docker-compose.voice.yml up -d
docker exec nlp2cmd-voice bash -c "export NLP2CMD_DEBUG=true && python -m nlp2cmd.service.docker_app"
```
